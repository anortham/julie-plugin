## Removed Julie Behavior Hooks From Plugin Distribution

- **WHAT:** Removed Claude/Codex/OpenCode behavior-hook distribution from `julie-plugin` while keeping the MCP launcher (`hooks/run.cjs`) and skills. `hooks/hooks.json` is now empty; Codex/OpenCode installers install skills only and prune legacy Julie hook/AGENTS artifacts on both install and uninstall.
- **WHY:** Hook-based coaching created config drift and local startup/debugging failures. Julie's stable distribution surface is MCP + skills; behavior hooks were not worth the operational overhead.
- **HOW:** Added `hooks/installers-no-hooks.test.cjs` red/green coverage, updated `bin/install-codex.cjs` and `bin/install-opencode.cjs`, deleted obsolete hook scripts/helpers/tests, cleaned `.gitattributes`, and updated README/CLAUDE docs.
- **IMPACT:** New installs no longer add behavior hooks. Re-running installers cleans old `julie-plugin` Codex hooks, old `julie-precedence` AGENTS blocks, and the OpenCode `plugins/julie-precedence.js` module without touching unrelated user config.
