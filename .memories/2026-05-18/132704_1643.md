## Fixed Plugin Launcher For Single-Binary v7.9.3 Archives

- **WHAT:** Audited `origin/main` after hook cleanup and found the remote was clean but the v7.9.3 release archives committed in `julie-plugin` still contain only `julie-server`, while the launcher expected `julie-adapter` and `julie-daemon`.
- **WHY:** Fresh plugin installs from current remote would extract the archive, fail to find `julie-adapter`, and fail before starting Julie.
- **HOW:** Updated `hooks/run.cjs` so `prepareBinaryForLaunch()` supports both archive shapes: split archives launch `julie-adapter`; legacy/single-binary archives fall back to `julie-server`. Added regression coverage in `hooks/run.test.cjs` and verified against the actual committed v7.9.3 Apple Silicon archive.
- **IMPACT:** Current remote can work with existing v7.9.3 archives, and future split archives still use the adapter path.
